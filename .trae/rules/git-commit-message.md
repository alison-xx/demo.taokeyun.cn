## 提交信息规范

### 格式要求

每次代码更新时，必须生成符合以下规范的提交信息：

```
<类型>: <简短描述>

<详细变动列表>
```

### 类型标签

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 重构（既不是新功能也不是修复）
- `perf`: 性能优化
- `test`: 添加/修改测试
- `chore`: 构建过程或辅助工具变动

### 示例

```
feat: 添加用户头像上传功能

- 新增头像上传组件
- 添加图片压缩处理
- 集成云存储服务
- 添加头像裁剪功能
```

## CHANGELOG 自动生成规则

### 触发条件

当检测到代码变更时，自动生成或更新 CHANGELOG.md 文件。

### 生成格式

```markdown
# Changelog

## [版本号] - 日期

### ✨ 新功能
- 功能描述

### 🐛 修复
- 修复描述

### 📝 文档
- 文档更新

### 其他变动
- 其他描述
```

### 更新流程

1. 分析本次提交的所有变更文件
2. 根据文件变更内容分类（前端/后端/配置/文档）
3. 提取关键改动点
4. 自动追加到 CHANGELOG.md 顶部
5. 提交时包含 CHANGELOG.md 的更新
